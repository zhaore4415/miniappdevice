整体流程就是:
1，有一个设备要寄出，在电脑端输入寄出地址与当前时间
2，用扫码枪扫描二维码，在后端记录这个信息（哪个设备，设备名称，设备型号，sn号，负责人等信，发到哪里，什么时间）
3，等用完之后发回来，然后再用扫码枪扫一下，去除地址与时间，归还设备
4在这期间可以在手机小程序上查看对应信息，查看设备是否有人使用（手机端并非扫码看寄出地址等，是做个标签这种形式可以直接点击看）

下面是我整理的，看一下，有没有补充的，技术用uniapp+.net core
[设备] 
  └─ 贴一个二维码（内容仅为 SN 号，如：SN123456）  ----这里要做个批量生成二维码的功能，支持下载和打印 ，你就得到很多纸质的二维码

[电脑端操作员]
  ├─ 打开浏览器 → 进入寄出登记页
  ├─ 输入设备名称，设备型号，sn号，负责人、「寄出地址」+「时间」（可自动填当前时间）
  └─ 用扫码枪扫设备二维码 → 系统自动提交：设备 SN123456 寄出！

[设备归还时]
  ├─ 回到同一页面（不填地址）
  └─ 再次用扫码枪扫 SN123456 → 系统自动标记为“已归还”

[手机用户]
  ├─ 打开手机微信小程序  
  ├─ 手机端并非扫码看寄出地址等，是做个标签这种形式可以直接点击看）
  └─ 立即看到：当前状态（使用中/空闲）、寄出地址、时间等
  
 

说明：
appdevice 这个小程序目录 用的vue3.0
webdevice 是后端目录，用.net core 8.0
